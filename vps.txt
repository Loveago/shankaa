
# VPS Deployment Guide for Tsk5 Backend and Frontend

This guide will walk you through deploying your Tsk5 backend and frontend applications on an Ubuntu VPS, using Nginx as a reverse proxy and pm2 to manage your Node.js applications. It assumes you have a Cloudflare domain pointed to your VPS IP.

**IMPORTANT:** Replace `your_domain.com`, `YOUR_VPS_IP`, and other placeholders with your actual values.

---

## Part 1: Initial VPS Setup (SSH into your VPS)

First, connect to your VPS via SSH.

```bash
ssh your_username@YOUR_VPS_IP
```

### 1. Update and Upgrade System Packages

It's good practice to start with an up-to-date system.

```bash
sudo apt update && sudo apt upgrade -y
```

### 2. Install Git, Nginx, and pm2

We'll need Git to clone your repositories, Nginx for serving the frontend and proxying backend requests, and pm2 to keep your Node.js applications running continuously.

```bash
sudo apt install git nginx -y
sudo npm install -g pm2 # We'll install npm shortly if not already present
```

### 3. Install Node.js and npm (Using NVM - Node Version Manager)

Using NVM is recommended for managing Node.js versions.

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
```
After running the above command, **you must close and re-open your SSH terminal** for NVM to be loaded into your shell.

Once reconnected, install a stable Node.js version:

```bash
nvm install --lts
nvm use --lts
nvm alias default 'lts/*' # Set LTS as default for new shells
```

### 4. Configure Firewall (UFW - Uncomplicated Firewall)

Open necessary ports: SSH (22), HTTP (80), HTTPS (443).

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx HTTP'
sudo ufw allow 'Nginx HTTPS' # We'll enable HTTPS later with Certbot
sudo ufw enable
sudo ufw status # Verify rules
```
Type `y` and press Enter when prompted to proceed with enabling the firewall.

---

## Part 2: Cloudflare Domain Setup (If not already done)

Ensure your domain is correctly configured in Cloudflare.

### 1. Add A Records

*   Go to your Cloudflare dashboard -> DNS -> Records.
*   Add an `A` record for your main domain:
    *   **Type:** `A`
    *   **Name:** `@` (or `your_domain.com`)
    *   **IPv4 address:** `YOUR_VPS_IP`
    *   **Proxy status:** `Proxied` (orange cloud)

*   (Optional but recommended) Add an `A` record for `www`:
    *   **Type:** `A`
    *   **Name:** `www`
    *   **IPv4 address:** `YOUR_VPS_IP`
    *   **Proxy status:** `Proxied` (orange cloud)

## Part 3: Deploying Your Applications

### 1. Clone Your Repositories

Create a directory for your projects and clone your frontend and backend repos into it.

```bash
mkdir ~/tsk5_projects
cd ~/tsk5_projects
git clone YOUR_FRONTEND_REPO_URL tsk5_frontend
git clone YOUR_BACKEND_REPO_URL tsk5_backend
```
Replace `YOUR_FRONTEND_REPO_URL` and `YOUR_BACKEND_REPO_URL` with the actual Git URLs of your repositories.

### 2. Install Dependencies and Build Frontend

#### Backend

```bash
cd ~/tsk5_projects/tsk5_backend
npm install
```

#### Frontend

```bash
cd ~/tsk5_projects/tsk5_frontend
npm install
```

### 3. Configure Backend Environment Variables

Create a `.env` file for your backend. This should contain your database connection string, JWT secrets, and any other sensitive information.

```bash
cd ~/tsk5_projects/tsk5_backend
nano .env
```
Paste your environment variables (from `tsk5_backend/.env.example` or your local setup) into this file.
**Crucially, add `FRONTEND_URL` for CORS configuration.**

```
DATABASE_URL="your_prisma_database_url_here"
JWT_SECRET="a_strong_random_secret_here"
# ... other backend specific env vars ...
PORT=5000 # Keep this as 5000 if you plan to use Nginx as configured later
FRONTEND_URL=http://your_domain.com # IMPORTANT: This will be updated to HTTPS later
```
Save and exit (`Ctrl+X`, `Y`, Enter).

### 4. Start Backend with pm2

Use pm2 to start your backend application and keep it running in the background.

```bash
cd ~/tsk5_projects/tsk5_backend
pm2 start index.js --name "tsk5_backend"
pm2 save # Saves the process list so pm2 restarts on reboot
pm2 status # Check if your app is running
```

### 5. Configure Frontend Environment Variables and Build

The frontend needs to know where your backend API is. We'll set `REACT_APP_API_URL` to your domain.

```bash
cd ~/tsk5_projects/tsk5_frontend
nano .env
```
Paste the following:

```
REACT_APP_API_URL=http://your_domain.com # IMPORTANT: This will be updated to HTTPS later
# Ensure there are no trailing slashes if your API expects specific paths.
# If your backend API starts with /api/, this should be fine.
```
Save and exit (`Ctrl+X`, `Y`, Enter).

Now, build your frontend application for production:

```bash
npm run build
```
This will create a `build` folder containing your static frontend files.

## Part 4: Nginx Configuration

We'll configure Nginx to serve your static frontend files and proxy API/Socket.io requests to your running backend.

### 1. Create Nginx Server Block Configuration

```bash
sudo nano /etc/nginx/sites-available/your_domain.com
```
Paste the following configuration. Remember to replace `your_domain.com`, `/home/your_username/tsk5_projects/tsk5_frontend/build`, and `/home/your_username/tsk5_projects/tsk5_backend/index.js` placeholders.

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name your_domain.com www.your_domain.com;

    # Frontend static files
    root /home/your_username/tsk5_projects/tsk5_frontend/build; # <<-- ADJUST THIS PATH
    index index.html index.htm;

    location / {
        try_files $uri $uri/ /index.html; # For single-page applications
    }

    # Proxy API requests to the backend
    location /api/ {
        proxy_pass http://localhost:5000; # Backend running internally on port 5000
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Proxy Socket.IO connections (ensure your frontend connects to /socket.io/)
    location /socket.io/ {
        proxy_pass http://localhost:5000/socket.io/; # Backend running internally on port 5000
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
Save and exit (`Ctrl+X`, `Y`, Enter).

### 2. Enable the Nginx Site

Create a symbolic link to enable your configuration:

```bash
sudo ln -s /etc/nginx/sites-available/your_domain.com /etc/nginx/sites-enabled/
```

### 3. Test Nginx Configuration

```bash
sudo nginx -t
```
You should see `test is successful`. If not, review your Nginx config file for syntax errors.

### 4. Restart Nginx

Apply the new configuration:

```bash
sudo systemctl restart nginx
```

### 5. Adjust Backend CORS Policy (Crucial for Security)

Currently, your backend CORS is `origin: "*"`. We need to restrict it to your actual frontend domain.

```bash
nano ~/tsk5_projects/tsk5_backend/index.js
```
Find the `cors` configurations and modify them as follows:

```javascript
// ... existing code ...

// Define allowed origins based on your frontend's URL
const allowedOrigins = [
  'http://localhost:3000', // For local frontend development
  'http://your_domain.com', // IMPORTANT: We will update this to HTTPS later
  'http://www.your_domain.com', // If you use www
  // Add any other specific origins your frontend might use
].filter(Boolean); // Remove null/undefined entries

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: {
    origin: (origin, callback) => {
      if (!origin || allowedOrigins.includes(origin)) {
        callback(null, true);
      } else {
        console.log(`CORS Error: Origin ${origin} not allowed`);
        callback(new Error('Not allowed by CORS'));
      }
    },
    methods: ["GET", "POST"]
  }
});

// ... existing code ...

app.use(cors({
  origin: (origin, callback) => {
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      console.log(`CORS Error: Origin ${origin} not allowed`);
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true // If your frontend sends cookies/auth headers
}));

// ... rest of file ...
```
Save and exit (`Ctrl+X`, `Y`, Enter).

Now, restart your backend application for the CORS changes to take effect:

```bash
pm2 restart tsk5_backend
```

At this point, you should be able to access your frontend via `http://your_domain.com`, and it should communicate with your backend.

## Part 5: Secure Your Application with HTTPS (Certbot/Let's Encrypt)

Securing your application with an SSL certificate is highly recommended. Cloudflare can proxy SSL, but it's best to have an end-to-end SSL encryption.

### 1. Install Certbot

```bash
sudo apt install certbot python3-certbot-nginx -y
```

### 2. Obtain SSL Certificate

Certbot will automatically configure Nginx for you.

```bash
sudo certbot --nginx -d your_domain.com -d www.your_domain.com
```
*   Follow the prompts:
    *   Enter your email address.
    *   Agree to terms of service.
    *   Choose whether to redirect HTTP to HTTPS (recommended: `2` for redirect).

### 3. Verify Certbot Auto-Renewal

Certbot sets up a cron job to renew certificates automatically. You can test it:

```bash
sudo certbot renew --dry-run
```
If this runs without errors, your auto-renewal is correctly configured.

### 4. Update Frontend and Backend to use HTTPS

Now that Nginx is configured for HTTPS, you should update your frontend's API URL and backend's `FRONTEND_URL` environment variable.

#### Frontend:
```bash
cd ~/tsk5_projects/tsk5_frontend
nano .env
```
Change:
`REACT_APP_API_URL=http://your_domain.com`
To:
`REACT_APP_API_URL=https://your_domain.com`

Save and exit, then **rebuild your frontend**:
```bash
npm run build
```

#### Backend:
```bash
cd ~/tsk5_projects/tsk5_backend
nano .env
```
Change:
`FRONTEND_URL=http://your_domain.com`
To:
`FRONTEND_URL=https://your_domain.com`

Save and exit.

**Update the `allowedOrigins` in your `index.js` file as well:**
```bash
nano ~/tsk5_projects/tsk5_backend/index.js
```
Change:
`'http://your_domain.com',` and `'http://www.your_domain.com',`
To:
`'https://your_domain.com',` and `'https://www.your_domain.com',`

Save and exit.

### 5. Restart Applications

```bash
pm2 restart tsk5_backend
sudo systemctl restart nginx
```

You should now be able to access your application securely via `https://your_domain.com`!

## Part 6: Continuous Deployment (Optional, Advanced)

For more automated deployments, consider setting up a CI/CD pipeline (e.g., GitHub Actions, GitLab CI) that:
1.  Pulls the latest code.
2.  Installs dependencies.
3.  Builds the frontend.
4.  Restarts the backend.
5.  Copies the new frontend build to the Nginx directory.
